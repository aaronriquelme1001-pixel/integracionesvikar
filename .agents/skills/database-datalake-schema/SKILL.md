# Database Datalake Schema

## Purpose
The `global_telemetry_traffic` table serves as the immutable source of truth for all historical positions, enabling complex analytics (like Billing and Fatigue scores) and state recovery.

## Schema
- `imei`: Device identifier.
- `plate`: Vehicle license plate.
- `lat`, `lng`: Coordinates.
- `speed`: Speed in km/h.
- `dt_tracker`: Native timestamp generated by the GPS hardware.
- `client_source`: The resolved client name (Zero-Code).
- `event`: Harsh driving events (haccel, hbrake, fatigue).

## Startup State Hydration
**CRITICAL RULE**: The Node.js engine is ephemeral. When it restarts, it loses its memory (`lastDeviceTimestamps`). 
To prevent missing historical gaps that occurred while the engine was offline, `gpsServer.js` MUST execute:
`SELECT imei, MAX(dt_tracker) FROM global_telemetry_traffic` 
on startup to hydrate the memory. This guarantees the Backfiller AI will trigger correctly on the first poll after a restart.
